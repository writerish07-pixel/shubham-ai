--- DUPLICATE FILES (older versions replaced by optimized counterparts) ---

1. main.py (1474 lines) -> REPLACED BY main_optimized.py (now renamed to main.py)
   Reason: Original used blocking I/O with ThreadPoolExecutor for ALL operations.
   The optimized version is async-native, uses httpx with connection pooling,
   has streaming LLM support, and reduced timeouts. ~370 fewer lines.

2. agent.py (392 lines) -> REPLACED BY agent_optimized.py (now renamed to agent.py)
   Reason: Original had verbose ~4000-token system prompt, no hybrid model routing,
   no response validation, no talk ratio tracking. The optimized version has:
   - Hybrid model routing (fast vs smart model)
   - Response validation (_validate_response)
   - Talk ratio enforcement (30/70 rule)
   - 60% shorter system prompt (~1500 tokens)

3. config.py (107 lines) -> REPLACED BY config_optimized.py (now renamed to config.py)
   Reason: Original lacked turn-taking constants, latency tuning knobs, and
   hybrid model configuration. The optimized version adds:
   - END_OF_SPEECH_SILENCE_MS, MIN_SPEECH_BYTES, WS_AUDIO_BUFFER_THRESHOLD
   - LLM_MAX_TOKENS_FAST/SMART, LLM_MIN_TOKENS_FLOOR
   - STT_TIMEOUT_SEC, LLM_TIMEOUT_SEC, TTS_TIMEOUT_SEC
   - GROQ_FAST_MODEL, GROQ_SMART_MODEL
   All values are env-var overridable.

4. call_handler.py (158 lines) -> REPLACED BY call_handler_optimized.py (now renamed to call_handler.py)
   Reason: Original had no turn-taking state flags, no async speech processing,
   no intent detection integration, no talk ratio tracking. The optimized version adds:
   - is_user_speaking, speech_final, is_ai_speaking flags
   - Async process_customer_speech_async()
   - Intent detection before LLM call
   - Talk ratio logging on call end

5. voice.py (169 lines) -> REPLACED BY voice_optimized.py (now renamed to voice.py)
   Reason: Original used blocking `requests` library with no connection pooling.
   The optimized version uses:
   - httpx with persistent connection pooling (~200ms savings per request)
   - Async-native STT and TTS functions
   - Parallel TTS chunk processing
   - Reduced timeouts (15-20s -> 5-6s)
